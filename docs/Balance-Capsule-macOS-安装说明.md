# Balance Capsule 安装说明

版本：`1.2.15-mac.14 · macOS 26+`

1. 打开 `BalanceCapsule-1.2.15-mac.14-arm64.dmg`。
2. 将 **Balance Capsule.app** 拖入 **Applications**。
3. 本地包没有 Apple Developer ID 公证。首次启动请在 Finder 的“应用程序”中右键 **Balance Capsule**，选择“打开”，再确认一次。
4. 拖动收起的玻璃球，或拖动展开后的胶囊面板，即可改变屏幕位置；释放后自动吸附边缘并保存位置。
5. 双击收起状态的悬浮球可立即刷新额度与 Token 统计；刷新期间菜单会显示“正在刷新…”，不会重复启动并发请求。

## 实现说明

界面没有使用设计效果图或截图贴图。液体玻璃、动态液位、气泡、精密刻度、扫光、Agent 标签和额度进度条全部由 AppKit/Core Animation 实时生成。macOS 原生背景采样层会先经过 Core Image 位移滤镜，再执行系统模糊和饱和度处理。系统材质自带的灰色覆盖层已移除，只保留真实背景采样、折射、模糊、低强度色散和精细高光。

折射模型参考 MIT 许可的 `rdev/liquid-glass-react`：程序实时计算球体、液滴颈部和圆角面板的 SDF 位移图，只折射玻璃边缘并保持中心区域清晰，同时绘制轻微的青色/红色色散边缘。原项目许可已包含在应用资源中的 `THIRD-PARTY-NOTICES.md`。

常驻悬浮球为 62pt，展开窗口为 410×210pt。球体、连接区和详情面板现在使用同一个连续轮廓，不再叠加三块玻璃或显示接缝。展开动画采用独立时间段：前 24% 只做液面反应，接着 28% 展示液滴伸长与收腰，最后 48% 将胶囊同时变宽、变高并形成完整面板；内容在轮廓接近完成后才淡入。

mac.8 修复无边框非激活面板只能点击展开的问题：悬浮球和详情窗口现在显式接收鼠标移动事件，并使用 `mouseEntered` 与 `mouseMoved` 双重触发。动画期间设有悬停状态锁，连续移动鼠标不会反复重启动画。正常使用只需把鼠标滑到悬浮球上，无需点击。

mac.9 整理菜单状态：悬浮球显示时菜单提供“隐藏悬浮球”，隐藏后自动改为“显示悬浮球”，两种状态使用同一个切换命令。已移除失效的“悬浮球显示（5 小时/一周）”子菜单和“动画效果”菜单项；旧版保存的 5 小时显示模式会在升级后自动迁移为 Codex 一周额度。

mac.11 将“今日 / 本月 / 总计”改为真正的 Codex 登录账号统计：今日与本月来自官方账号的每日 Token 桶，总计来自账号累计 Token。旧版对本机 `~/.codex/sessions` 和 `archived_sessions` 的扫描已全部删除。双击悬浮球会同时刷新额度与账号统计；即使账号统计接口暂时延迟，已读取的额度仍会保留。

切换数据源后绝不混用数据：选择 Codex 时只显示当前 Codex 登录账号返回的 Token；选择 Claude Code 时不会显示 Codex 或本机估算值。Anthropic 目前只向组织提供 Claude Code Analytics API，个人 Pro/Max 账号没有同等的程序化账号 Token 接口；Team/Enterprise/Console 组织需要另行配置 Admin API Key 或 Analytics API Key。

mac.12 将应用图标外围重新绘制成高透蓝白液态玻璃，并使用完整的 16px–1024px `.icns` 多分辨率资源，避免 Finder 为旧透明 PNG 添加灰色底板。详情面板的百分号会根据数字实际宽度保持 4pt 间距；Token 数量只使用中文“万 / 亿”单位，例如 `24.8万`、`2.6亿`、`7.58亿`。

mac.13 修正 macOS 对透明图标自动补灰底的问题：液态玻璃现在完整铺满图标画布，再由系统统一裁切最外层轮廓，不会出现“灰色外框 + 蓝色内框”的双层方块；中央悬浮球同步放大。

## 支持范围

- macOS 26 或更新版本
- 仅支持 Apple Silicon（M1、M2、M3、M4 或后续 arm64 芯片），不支持 Intel Mac
- Codex 官方额度
- Codex 当前登录账号的今日、本月和累计 Token
- DeepSeek 账户余额
- OpenRouter 当前 API Key 剩余额度
- Claude Code 官方额度（通过菜单安装本地桥接）

程序不会读取聊天内容、本机会话日志、浏览器 Cookie 或 Codex `auth.json`。
