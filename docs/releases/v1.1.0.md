# Quota Orb v1.1.0

## 新增

- 自动识别官方 Codex、普通第三方配置和已验证的 CC Switch 本地代理。
- DeepSeek 账户余额，多币种原样显示。
- OpenRouter 当前推理 Key 的剩余额度上限，明确区别于账户总余额。
- CC Switch 3.16.5 只读当前 Codex 供应商、代理端口和内置余额模板。
- 兼容 Codex++ v1.2.36 的活动 Relay 状态；明确绑定 DeepSeek 或 OpenRouter 单一 HTTPS 上游时读取对应余额。
- Codex++ 聚合、轮转及未知 Relay 的安全“不支持”状态。

## 稳定性

- 刷新失败时保留同一来源的上次成功结果并标记为过期。
- 切换供应商时立即清空旧来源缓存，避免显示错误账户的余额。
- 继续使用一个刷新服务和一个周期循环；官方额度 15 秒刷新，第三方余额至少间隔 60 秒。

## 安全

- 余额请求仅允许 HTTPS、GET、同主机且禁止重定向。
- 默认超时 10 秒，响应体最大 256 KB。
- 凭据只在请求期间读取，不持久化、不记录、不显示。
- 不执行 CC Switch 自定义 JavaScript 用量脚本。
- Codex++ 无法安全确定单一上游账户时失败关闭。
- CC Switch 只读访问改用 Windows 系统 SQLite，跟随系统安全更新；使用时应保持 Windows Update 为最新状态。

## 当前限制

- 不聚合多个供应商余额。
- 不支持跨主机余额端点。
- 不查询需要 OpenRouter Management Key 的账户总余额接口。
- 未知供应商需要后续增加明确的原生适配器。
